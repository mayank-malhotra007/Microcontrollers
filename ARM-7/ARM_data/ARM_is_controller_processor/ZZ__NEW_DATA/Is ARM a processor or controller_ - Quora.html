Is ARM a processor or controller?

ARM is basically a family of architectures. There exist several variations of the basic architecture depending upon the applications for which they are targeted.

To correctly answer your question, lets recall what a micro-controller is. A micro-controller, to put in the easiest way, is a small processor combined with some memory, power and clock controlling logic, i/o controlling logic and some other supporting logic packaged into a single package. The small processor or the core here could be the one based on ARM.

Some high performance microprocessors are present in the market which are ARM based. Most of the processor cores used in the modern day mobile platforms like Tablets and Smartphones are either ARM or some derivative of the ARM architecture.

To determine which architecture is best suited for micro-controller core or for a microprocessor, depends upon the requirements of the design. For instance, micro-controllers are generally needed to be as power efficient as possible while performance can be compromised to some extent. On the other hand the cores running in server have to be heavy performers, power consumption is not that much of a stringent constraint. While the chips running inside your smartphone or tablets have to optimize on both the aspects.

Thus, there exist several variants of the basic ARM architecture. Like Cortex M series which is targeted for micro-controllers while the Cortex A series is more performance oriented.

Hope this clarifies your doubt.

I know exactly where you are getting confused as I had a similar doubt myself.

Well as mentioned earlier by others  ARM is an architecture. What is an architecture you may ask? Long story short, you can treat an architecture as a design  which been patented and  requires you to pay royalties in order to acquire it and to implement it in your system.

So since ARM is an architecture then does it not qualify as a controller nor a processor?  Well this is a bit tricky to answer. Lets start with a microcontroller. A microcontroller is nothing but a processor with RAM, ROM, IO ports, Timers all built onto one system. Its like a computer all set to go! But when we talk about a processor, we are talking only about the processor without all the rest of the stuff that a microcontroller has. A processor is not set to go and you would actually need to interface quite a lot of things around it to make something useful out of it.

So when we hear about a microcontroller based on the ARM architecture what it means is that the processor it contains has been built around the architecture implemented by ARM only after paying royalties for it. And similarly when you come across a ARM based standalone processor it means that it has also acquired the royalties to utilise the design.

So arm is neither a processor or a microcontroller in its purest form. Instead it a design on which these are based. And this design can be utilised by anyone willing to pay the royalties.

I hope this helps you!
